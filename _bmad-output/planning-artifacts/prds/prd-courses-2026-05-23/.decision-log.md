# Decision Log - Attendance Document Workflow System PRD

## Decisions Made

### 2026-05-23 - Initial Requirements Capture

**Decision:** Scope PRD to attendance document workflow with custom workflow support
**Rationale:** User clarified primary use case is attendance, but also needs custom document approval capability
**Impact:** Features F1-F5 focus on attendance, F6 covers custom workflows

**Decision:** Use MinIO `lms-workflow` bucket for document storage
**Rationale:** Leverages existing MinIO infrastructure, avoids Nextcloud dependency
**Impact:** Document storage integrated with Smart Drive system

**Decision:** Document submission from attendance screen, not Smart Drive
**Rationale:** Instructors may not have Smart Drive access; attendance screen is natural workflow entry point
**Impact:** F1.1-F1.8 focus on attendance screen integration

**Decision:** Full version history using MinIO native versioning
**Rationale:** User explicitly requested Google Docs-style version history for audit trail
**Impact:** F3 covers comprehensive versioning and change tracking

**Decision:** Status flow: Draft → Submitted → Under Review → Approved/Rejected → Amended → Closed
**Rationale:** User approved this flow with requirement for multiple review cycles
**Impact:** F7 defines status transitions and tracking

**Decision:** HR can amend attendance and regenerate document as source of truth
**Rationale:** User needs HR to have authority to amend and update documents directly
**Impact:** F4 covers amendment workflow and regeneration

**Decision:** Support both system exports and manual uploads (scanned documents)
**Rationale:** User mentioned scanning capability for flexibility
**Impact:** F1.8 includes manual upload support

**Decision:** Comments visible to all roles (not just HR/Admin)
**Rationale:** User confirmed "yes" when asked about comment visibility
**Impact:** FR-2.7, FR-2.8, FR-2.9 specify comment visibility

**Decision:** Structured document naming convention linking to class/instructor/date/program/subject
**Rationale:** User requested dynamic convention for document organization
**Impact:** F1.5 defines naming convention with metadata linkage

## Resolved Questions (2026-05-23)

1. **Scanned documents:** Treated as static attachments, no OCR processing
2. **Maximum review cycles:** 3 cycles before HR can force-close
3. **Instructor withdrawal:** Yes, instructors can withdraw before HR review
4. **Automatic archival:** Not implementing (deferred for future)
5. **HR delegation:** Role-based - any user with HR role can review
6. **SLA requirements:** 72 hours with color coding and inbox ordering
7. **Optional attachments:** Custom workflows can trigger without file attachments
