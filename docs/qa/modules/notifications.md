# Notifications Module

## Business Context
Notifications alert users to system events: new announcements, chat messages, workflow approvals, attendance reminders. Users can mark read/unread, archive, and configure preferences (email/push). Admin can send test notifications.

## API Routes
| Method | Path | Description | Roles |
|--------|------|-------------|-------|
| GET | `/api/v1/notifications` | List | all authenticated |
| PATCH | `/api/v1/notifications/:id/read` | Mark read | all authenticated |
| PATCH | `/api/v1/notifications/:id/unread` | Mark unread | all authenticated |
| POST | `/api/v1/notifications/mark-all-read` | Mark all read | all authenticated |
| PATCH | `/api/v1/notifications/:id/archive` | Archive | all authenticated |
| POST | `/api/v1/notifications/archive-all-read` | Archive all read | all authenticated |
| DELETE | `/api/v1/notifications/:id` | Delete | all authenticated |
| GET | `/api/v1/notifications/preferences` | Get preferences | all authenticated |
| PUT | `/api/v1/notifications/preferences` | Update preferences | all authenticated |
| POST | `/api/v1/notifications/admin/test` | Test notification | admin, super_admin |

## UI Pages
- Notification drawer in header (bell icon)

## Business Rules
- Notifications generated by: announcements, chat, workflow, attendance
- User preferences: email notifications, push notifications
- Archive system for read notifications
- Admin test endpoint for verifying delivery
- Real-time delivery via WebSocket
