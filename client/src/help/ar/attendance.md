# الحضور

شاشة الحضور تتيح للمدربين وموظفي الموارد البشرية تسجيل ومراجعة وتصدير سجلات الحضور. تتضمن أيضاً ماسح QR لتسجيل دخول سريع.

## من يمكنه الوصول

| الدور | العمليات | ما يمكنهم فعله |
| --- | --- | --- |
| المشرف العام | عرض، إنشاء، تحديث، حذف | إدارة كاملة للحضور |
| المشرف | عرض، إنشاء، تحديث، حذف | إدارة كاملة للحضور |
| الموارد البشرية | عرض، إنشاء، تحديث، حذف | حضور الموارد البشرية والعمليات بالجملة |
| المدرب | عرض، إنشاء، تحديث | تسجيل الحضور لفصولهم |
| الطالب | عرض | عرض سجلات الحضور الخاصة بهم |

> **معرفات الشاشات:** `attendance` (رئيسي)، `hr-attendance` (موارد بشرية بالجملة)، `qr-scanner` (مسح يومي). لكل منها فحوصات صلاحيات منفصلة.

## الإجراءات الرئيسية

- **تسجيل الحضور** — وضع علامة للطلاب كحاضر أو غائب أو متأخر أو بعذر لجلسة فصل محددة.
- **ماسح QR** — فتح شاشة ماسح QR لتسجيل دخول الطلاب بمسح رمز QR الخاص بهم. يتطلب صلاحية `qr-scanner.canUseQRScanner`.
- **حضور الموارد البشرية** — تسجيل الحضور بالجملة لأغراض الموارد البشرية. يُaccess عبر شاشة حضور الموارد البشرية.
- **تصدير** — تنزيل تقارير الحضور كملف Excel أو CSV. يتطلب صلاحية `attendance.canExport` أو `qr-scanner.canExport`.
- **تحرير السجلات** — تعديل حالة الحضور بعد الجلسة. يتطلب صلاحية `update`.

## حالات الحضور

| الحالة | الوصف |
| --- | --- |
| **حاضر** | حضر الطالب الجلسة. |
| **غائب** | لم يحضر الطالب. |
| **متأخر** | وصل الطالب بعد وقت البداية. |
| **بعذر** | غياب الطالب بعذر مُوافق عليه. |

## صلاحيات ماسح QR التفصيلية

ماسح QR لديه مجموعة خاصة من العمليات التفصيلية:

| العملية | الوصف |
| --- | --- |
| `canMarkAttendance` | تسجيل الحضور عبر الماسح |
| `canUseQRScanner` | الوصول إلى واجهة ماسح QR |
| `canManualInput` | إدخال الحضور يدوياً دون مسح |
| `canEditAttendance` | تحرير سجلات الحضور لليوم |
| `canDeleteAttendance` | حذف سجلات الحضور |
| `canBulkScan` | مسح عدة طلاب بالتتابع |
| `canExport` | تصدير بيانات الحضور |
| `canExportSummary` | تصدير تقرير ملخص |

## القواعد والتحقق

- **جلسة مطلوبة** — يجب اختيار جلسة فصل قبل تسجيل الحضور.
- **التحقق من التاريخ** — يمكن تسجيل الحضور فقط للتاريخ الحالي أو التواريخ السابقة (ضمن نافذة قابلة للتحرير). التواريخ المستقبلية محظورة.
- **سجل واحد لكل طالب لكل جلسة** — كل طالب يمكن أن يكون له سجل حضور واحد فقط لكل جلسة. السجلات المكررة تُرفض.
- **سجل تغيير الحالة** — جميع تعديلات الحضور تُسجل مع المستخدم والطابع الزمني والقيمة السابقة لأغراض التدقيق.
- **الحضور يُغذي العقوبات** — الغياب والتأخير يمكن أن تولد تلقائياً سجلات عقوبات عبر القواعد المُكونة.
- **الحضور يُغذي المشاركة** — الحضور والتسجيل في الوقت يساهم في درجات المشاركة.

## القيود

- ماسح QR يتطلب الوصول إلى الكاميرا. إذا منع المتصفح أذونات الكاميرا، الإدخال اليدوي هو البديل.
- حضور الموارد البشرية بالجملة لا يمكن التراجع عنه في إجراء واحد — كل سجل يجب تحريره فردياً.
- تصدير الحضور محدود بـ 10,000 سجل لكل طلب. للتصديرات الأكبر، قلل نطاق التاريخ.
