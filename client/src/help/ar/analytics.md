# التحليلات

تجربة التحليلات عبارة عن نظام لوحة معلومات مرن يعتمد على الأدوات. يمكنك بناء وترتيب بطاقات الرسوم البيانية والقوائم ومؤشرات القيمة الفردية لتصور البيانات المهمة لدورك. نفس محرك الأدوات يشغل شاشة **التحليلات المتقدمة**، وقسم **لوحة الملخص → التحليلات**، وألوية التحليلات داخل **لوحة تحكم الطالب** والشاشات الأخرى.

شاشة التحليلات تتكون من منطقتين رئيسيتين:

1. **إحصائيات لوحة التحكم** — بطاقات ملخص تعرض أعداداً رئيسية (البرامج، المواد، الصفوف، التسجيلات، الأنشطة، الموارد، المستخدمين حسب الدور، الاختبارات، الإعلانات، العقوبات، السلوكيات، المشاركات) مع إمكانية التصفية حسب البرنامج والمادة والصف.
2. **لوحة تحليلات لوحة التحكم** — شبكة أدوات قابلة للسحب وتغيير الحجم تغطي فئات القرص الذكي وسير العمل والأنشطة.

## من يمكنه الوصول

| الدور | الوصول |
| --- | --- |
| المشرف العام | تحليلات كاملة، وجميع الأدوات، والتصدير، وتحرير التخطيط. يرى أعداداً على مستوى النظام بأكمله. |
| المشرف | تحليلات كاملة ضمن البرامج المتاحة. يرى أعداداً للبرامج المتاحة له. |
| الموارد البشرية | تحليلات المستخدمين والحضور. |
| المدرب | صفوفه الشخصية، والأنشطة، والحضور، وتحليلات الدرجات. يرى صفوفه فقط والبيانات المرتبطة بها. |
| الطالب | تحليلات محدودة أو للقراءة فقط حسب إعدادات الدور. |

## معرّفات الشاشات

- `advancedAnalytics` — صفحة `/advanced-analytics`
- `summary-dashboard` — صفحة لوحة الملخص مع أقسام التحليلات
- `dashboard` — ألوية تحليلات لوحة التحكم (حسب الدور)

## إحصائيات لوحة التحكم (بطاقات الملخص)

القسم العلوي من صفحة التحليلات يعرض مجموعة من بطاقات الملخص داخل قسم قابل للطي بعنوان **إحصائيات لوحة التحكم**. كل بطاقة تعرض عدداً مع أيقونة وتسمية وتلميح.

### المرشحات

أعلى بطاقات الملخص، يوجد شريط تصفية يتيح تضييق جميع الإحصائيات:

- **البرنامج** — اختر برنامجاً لتصفية المواد والصفوف والتسجيلات والأنشطة والموارد حسب ذلك البرنامج.
- **المادة** — اختر مادة (مصفاة حسب البرنامج المختار) لمزيد من التضييق للصفوف والتسجيلات والأنشطة.
- **الصف** — اختر صفًا محدداً لرؤية تسجيلاته وأنشطته وإعلاناته فقط.

تغيير مرشح البرنامج يعيد تعيين مرشحي المادة والصف. تغيير مرشح المادة يعيد تعيين مرشح الصف. يعاد جلب عدد الموارد من الخادم عند تغيير أي مرشح.

### بطاقات الملخص (الصف الأول)

| البطاقة | من يراها | ما تعرضه |
| --- | --- | --- |
| **البرامج** | المشرف العام فقط | إجمالي عدد البرامج في النظام. |
| **المواد** | المشرف والمشرف العام | المواد في النظام (المشرف العام) أو في البرامج المتاحة (المشرف). مصفاة حسب البرنامج إذا تم اختياره. |
| **الصفوف** | جميع الأدوار | الصفوف مصفاة حسب البرنامج/المادة/الصف. المدربون يرون صفوفهم فقط (مطابقة بـ `instructorId` أو `ownerEmail` أو حقل `instructor`). |
| **التسجيلات** | جميع الأدوار | التسجيلات مصفاة حسب الصف/المادة/البرنامج. المدربون يرون تسجيلات صفوفهم فقط. |
| **الأنشطة** | جميع الأدوار | الأنشطة مصفاة حسب الصف/المادة/البرنامج. المدربون يرون أنشطة صفوفهم فقط. |
| **الموارد** | جميع الأدوار | عدد الموارد من الخادم مصفى حسب البرنامج/المادة/الصف. يعرض `...` أثناء التحميل. |

### بطاقات أدوار المستخدمين (الصف الثاني)

مرئية لـ **المشرف والمشرف العام فقط**. تعرض أعداداً لكل دور مستخدم:

- **الطلاب** — المستخدمون الذين لديهم `isStudent === true`
- **المدربون** — المستخدمون الذين لديهم `isInstructor === true`
- **الموارد البشرية** — المستخدمون الذين لديهم `isHR === true`
- **المشرفون** — المستخدمون الذين لديهم `isAdmin === true`
- **المشرفون العامون** — المستخدمون الذين لديهم `isSuperAdmin === true`

### بطاقات الأنشطة (الصف الثالث)

مرئية لـ **جميع الأدوار**. تعرض أعداداً لـ:

- **الأنشطة** — مصفاة حسب الصف/المادة/البرنامج. المدربون يرون أنشطتهم فقط.
- **الاختبارات** — إجمالي الاختبارات. النقر على البطاقة ينقل إلى `/quizzes`.
- **الإعلانات** — مصفاة حسب الصف/المادة/البرنامج.
- **العقوبات** — مصفاة حسب الصف/المادة/البرنامج. المدربون يرون عقوباتهم فقط.
- **السلوكيات** — مصفاة حسب الصف/المادة/البرنامج. المدربون يرون سلوكياتهم فقط.
- **المشاركات** — مصفاة حسب الصف/المادة/البرنامج. المدربون يرون مشاركاتهم فقط.

### تحميل البيانات

جميع بيانات بطاقات الملخص تُحمّل بالتوازي باستخدام `Promise.allSettled` عند تحميل المكوّن. إذا فشل أحد الخدمات، تعرض البطاقة `0` ويُسجل تحذير. زر التحديث في رأس القسم يعيد جلب جميع البيانات ويعرض رسالة نجاح/فشل.

### الجولة الموجهة

قسم إحصائيات لوحة التحكم يتضمن **جولة موجهة** (مدعومة بـ Joyride) تبدأ تلقائياً عند الزيارة الأولى. تغطي الجولة:

1. **المرشحات** — كيفية استخدام مرشحات البرنامج/المادة/الصف.
2. **بطاقات الملخص** — الأعداد الرئيسية للبرامج والصفوف والتسجيلات والأنشطة والموارد.
3. **بطاقات أدوار المستخدمين** — أعداد المستخدمين حسب الدور (الطلاب، المدربون، الموارد البشرية، المشرفون، المشرفون العامون).
4. **بطاقات الأنشطة** — إجماليات الاختبارات والإعلانات والعقوبات والسلوكيات والمشاركات.
5. **التحديث** — كيفية تحديث لوحة التحكم بأحدث البيانات.

يمكن إعادة تشغيل الجولة بالنقر على **أيقونة المساعدة** (?) في رأس القسم. يُحفظ إكمال الجولة في `localStorage` لكل لغة (`analyticsDashboardTourSeen_en` / `analyticsDashboardTourSeen_ar`).

## لوحة تحليلات لوحة التحكم

أسفل بطاقات الملخص توجد **لوحة تحليلات لوحة التحكم** — شبكة أدوات مدعومة بمكوّن `DashboardEngine`. توفر هذه اللوحة أدوات جاهزة ومخصصة تغطي ثلاث فئات:

- **القرص الذكي** — أعداد الملفات، استخدام التخزين، نشاطات الملفات، الملفات حسب النوع/الدلو، الملفات الحديثة.
- **سير العمل** — أعداد المستندات حسب الحالة/النوع/البرنامج، معدل الموافقة، الخط الزمني، المستندات الحديثة.
- **الأنشطة** — أعداد الأنشطة/الإرسالات/الموارد، الأنشطة حسب النوع، الخطوط الزمنية، الأنشطة/الإرسالات الحديثة.

## مكونات شريط الأدوات

يقع شريط الأدوات أعلى شبكة الأدوات. كل أيقونة تنفذ إجراءً محدداً.

| الأيقونة / الزر | الإجراء | ما يفعله |
| --- | --- | --- |
| **البحث في الأدوات** | تصفية | اكتب لتضييق الأدوات المرئية حسب العنوان (إنجليزي، عربي)، المعرف، أو حقل التجميع. تبقى فقط البطاقات المطابقة. |
| **تبويبات الفئات** | تصفية | اختر فئة: الكل، القرص الذكي، سير العمل، أو الأنشطة. تُعرض الأدوات في هذه الفئة فقط. تستخدم `inferAnalyticsWidgetCategory` لتعيين كل مصدر بيانات إلى فئته. |
| **تحديث** | إعادة تحميل | يعيد جلب البيانات التحليلية الأساسية من واجهة برمجة الخادم (`DashboardAnalyticsService`) دون إعادة تحميل الصفحة. |
| **تحرير التخطيط** | تبديل الوضع | يتيح السحب والإفلات ومقابض تغيير الحجم لكل أداة. انقل البطاقات، غيّر حجمها، ثم انقر الزر مرة أخرى لقفل التخطيط. |
| **إضافة أداة** | فتح المنشئ | يفتح نموذج بناء الأدوات لإنشاء رسم بياني أو قائمة أو بطاقة عدد جديدة. يراعي الحد الأقصى لعدد الأدوات. |
| **إعادة التعيين** | استعادة الافتراضيات | يفتح نافذة تأكيد. عند التأكيد، يزيل جميع الأدوات المخصصة ويستعيد مجموعة الأدوات الافتراضية للنظام. |

## إجراءات التمرير فوق الأداة

مرّر فوق أي بطاقة أداة لإظهار شريط إجراءات صغير في الزاوية العلوية اليمنى.

| الأيقونة | الإجراء | الوصف |
| --- | --- | --- |
| السهم للأعلى/للأسفل | **تصغير / استعادة** | طي الأداة لإظهار الرأس فقط (الارتفاع = صف واحد)، أو إعادة توسيعها إلى حجمها الأصلي. يتم حفظ الحجم الأصلي واستعادته. |
| القلم | **تحرير** | إعادة فتح نموذج بناء الأدوات لهذه الأداة مع جميع إعداداتها الحالية. |
| النسخ | **تكرار** | إنشاء نسخة مطابقة من الأداة بمعرف جديد ولاحقة "(نسخة)" في العنوان. مزاحة بمقدار موضع واحد عن الأصل. تراعي حد الأدوات. |
| التحميل | **حفظ الرسم** | حفظ الرسم البياني الحالي كملف SVG. مخفي لأدوات القائمة والعدد. |
| سلة المهملات | **حذف** | إزالة الأداة من لوحة التحكم فوراً. ينظّف أيضاً حالة التصغير وإصدارات التحديث والطوابع الزمنية. |
| تحديث (لكل أداة) | **تحديث الأداة** | يزيد عداد إصدار الأداة لإعادة الرسم من البيانات المحملة بالفعل. إذا كان `onSmartReload` متاحاً، يجلب بيانات حديثة لتلك الأداة. يعرض مؤشر "تم التحديث حديثاً" لمدة ~1.4 ثانية. |

## تكبير الأداة

النقر على رأس الأداة (أو زر التكبير) يفتح **نافذة ملء الشاشة** تعرض الرسم بأقصى حجم. مفيد للفحص التفصيلي للرسوم والقوائم المعقدة.

## إضافة / تحرير أداة

انقر **إضافة أداة** أو مرّر فوق بطاقة وانقر **تحرير** لفتح نموذج بناء الأدوات. يحتوي النموذج على الحقول التالية:

### العنوان

- **العنوان** — عنوان اختياري يظهر على البطاقة.
- **العنوان (بالإنجليزية)** — النسخة الإنجليزية من العنوان (`titleEn`).
- **العنوان (بالعربية)** — النسخة العربية من العنوان (`titleAr`). يُعرض العنوان الصحيح تلقائياً بناءً على اللغة المختارة (`getWidgetDisplayTitle`).

### نوع الرسم البياني

اختر نوع التصور. ستة خيارات متاحة كأزرار أيقونات:

| النوع | الأيقونة | الأفضل لـ |
| --- | --- | --- |
| **شريطي** | رسم شريطي | مقارنة القيم عبر الفئات. |
| **خطي** | رسم خطي | إظهار الاتجاهات عبر الزمن. |
| **دائري** | رسم دائري | إظهار أجزاء من الكل. |
| **حلقي** | رسم حلقي | تصور مدمج لأجزاء من الكل. |
| **قائمة** | أيقونة قائمة | عرض الصفوف الخام مع حد صفّي قابل للضبط. |
| **عدد** | أيقونة هاش | إبراز مقياس رقمي واحد. |

لا يتوافر كل مصدر بيانات مع كل نوع رسم. يُعطل المنشئ الخيارات غير المتوافقة (تُعرض بشفافية 35% ومؤشر `not-allowed`). المصادر المتوافقة تحدد بـ `getSourcesForChartType(chartType, category)`.

### مصدر البيانات

اختر مصدر البيانات الأساسي من قائمة منسدلة. تعتمد المصادر المتاحة على تبويب الفئة المختار ونوع الرسم. تبويبات الفئات في أعلى قسم مصدر البيانات تتيح التبديل بين:

- **الكل** — جميع الفئات
- **أكاديمي** — الأنشطة، الإعلانات، الموارد، التسجيلات، الصفوف، البرامج، المواد
- **تقييم** — درجات الطلاب، الاختبارات، إرسالات الاختبارات
- **عمليات** — الحضور، المستخدمون، العقوبات، الغيابات، السلوكيات، المشاركات، سجلات النشاط
- **محتوى** — ملفات القرص الذكي، مشاركات القرص الذكي
- **سير العمل** — مستندات سير العمل، مهام سير العمل
- **جدولة** — الجلسات، المدربون، الصفوف، القاعات، الاستراحات، العطل، الحضور، سير العمل
- **طالب** — نظرة عامة على الطالب، الحضور، العقوبات، السلوكيات، المشاركات، الدرجات، التسجيلات
- **تحليلات** — نظرة عامة على القرص الذكي، نظرة عامة على سير العمل، نظرة عامة على الأنشطة، وتفاصيلها الدقيقة

عند تبديل الفئات، تتحدث قائمة مصدر البيانات تلقائياً. إذا لم يكن المصدر الحالي متاحاً في الفئة الجديدة، يُعين إلى أول مصدر متاح.

### ما الذي سيتم عدّه (أدوات العدد)

عندما يكون نوع الرسم **عدد**، تظهر شبكة من أزرار المقاييس. كل مقياس يعرض عدداً محسوباً مسبقاً مختلفاً. على سبيل المثال:

- **نظرة عامة على القرص الذكي**: إجمالي الملفات، إجمالي المجلدات، إجمالي التخزين، نشاطات الملفات
- **نظرة عامة على سير العمل**: إجمالي المستندات، الموافق عليها، المعلقة، المرفوضة، معدل الموافقة
- **نظرة عامة على الأنشطة**: إجمالي الأنشطة، إجمالي الإرسالات، إجمالي الموارد

اختيار مقياس يضبط `countMetric` و`statKey` و`titleKey` ويمسح `groupBy`.

### المقياس (المصادر الرقمية)

بالنسبة للمصادر التي تعرض حقولاً رقمية للقيمة، اختر **المقياس** الذي تريد تجميعه:

- `sessionCount` — عدد الجلسات
- `teachingHours` — إجمالي ساعات التدريس
- `fileCount` — عدد الملفات
- `activityCount` — عدد الأنشطة
- `documentCount` — عدد المستندات
- `submissionCount` — عدد الإرسالات
- `resourceCount` — عدد الموارد
- `storageMB` — التخزين بالميجابايت
- `score` — درجة الاختبار
- حقول رقمية أخرى حسب المصدر

يتم تعيين التجميع افتراضياً إلى **المجموع** للمقاييس الرقمية. بالنسبة لمصادر العدد البسيط، يكون التجميع دائماً **العدد**.

### التجميع حسب

بالنسبة للرسوم البيانية الشريطية والخطية والدائرية والحلقية، اختر كيفية تجميع البيانات. تُعرض شبكة من الأزرار تتضمن خيار **لا شيء**. الخيارات الشائعة تشمل:

- لا شيء
- الحالة / النوع
- البرنامج / المادة / الصف
- المدرب / الطالب / القاعة
- التاريخ / الفصل / السنة
- نوع العقوبة / نوع الحضور / نوع الاستراحة
- نوع MIME / الدلو / الإجراء
- نوع سير العمل / نوع النشاط / نوع المورد

تعتمد الخيارات المتاحة على إعداد `groupBy` لمصدر البيانات المختار.

### التجميع

بالنسبة للمصادر بدون حقول قيمة رقمية، التجميع هو **العدد** فقط. للمصادر ذات الحقول الرقمية، ثلاثة خيارات متاحة:

- **العدد** — عدد السجلات
- **المجموع** — إجمالي حقل القيمة
- **المتوسط** — متوسط حقل القيمة

### نطاق التاريخ

قم بتقييد البيانات لفترة. ستة أزرار مسبقة تُعرض:

| النطاق | الوصف |
| --- | --- |
| **الكل** | جميع البيانات المتاحة |
| **اليوم** | بيانات اليوم فقط |
| **آخر 7 أيام** | نافذة متحركة لمدة 7 أيام |
| **آخر 30 يوماً** | نافذة متحركة لمدة 30 يوماً |
| **آخر 90 يوماً** | نافذة متحركة لمدة 90 يوماً |
| **نطاق مخصص** | اختيار تواريخ بداية ونهاية صريحة باستخدام منزلق نطاق التاريخ |

عند اختيار **نطاق مخصص**، يظهر مكوّن `DateRangeSlider` مع منتقيات تاريخ من/إلى.

### حد الصفوف (أدوات القائمة)

بالنسبة لأدوات القائمة، اختر عدد الصفوف المراد عرضها: **10، 25، 50، 100، أو 200**. الافتراضي هو 50.

### حجم التخطيط

- **العرض (أعمدة 2–12)** — عدد أعمدة الشبكة التي تشغلها الأداة. الصف الكامل يساوي 12 عموداً. الافتراضي هو 6.
- **الارتفاع (صفوف 2–12)** — عدد الصفوف الرأسية التي تشغلها الأداة. الافتراضي هو 4.

### حفظ أو تحديث

- **إنشاء** — إضافة الأداة إلى لوحة التحكم بمعرف فريد (`w` + طابع زمني).
- **تحديث** — حفظ التغييرات في الأداة الموجودة.
- **إلغاء** — إغلاق المنشئ دون حفظ.

## عرض الرسوم البيانية

تُعرض الرسوم البيانية بواسطة مكوّن `OptimizedChartRenderer`، الذي يستخدم **React.lazy** و**Suspense** لتحميل أنواع الرسوم (شريطي، خطي، دائري، قائمة) عند الطلب، مما يقلل حجم الحزمة الأولي. المحرك مُحفوظ (memoized) لمنع إعادة العرض غير الضرورية.

### عرض أداة العدد

تعرض أدوات العدد رقماً كبيراً في وسط البطاقة مع:

- خلفية متدرجة شعاعية متحركة (تأثير نبض)
- ظهور متحرك للرقم (تحريك + تكبير)
- حجم خط متجاوب حسب أبعاد الأداة (2.5rem–7rem)
- خلفية وحدود متدرجة بلون مميز
- رقم منسق بـ `toLocaleString()`

### عرض أداة القائمة

تعرض أدوات القائمة صفوف بيانات خام مع:

- حد صفوف قابل للضبط
- إدارة الأعمدة (إضافة/إزالة أعمدة)
- بيانات من مصفوفات التحليلات الخام
- تسميات حالة ونوع طبيعية (عبر `normalizeAttendanceStatus` و`normalizeActivityType`)

## التفاعلات التنقلية التفصيلية (Drill-down)

النقر على نقطة بيانات في رسم بياني (شريحة دائرية، مقطع شريطي، نقطة خطية) يُطلق **تنقّلاً تفصيلياً**:

1. يتحقق النظام مما إذا كانت هناك أداة قائمة تنقل تفصيلي محددة مسبقاً لمصدر البيانات ونقطة البيانات المنقورة.
2. إذا وُجدت، تُضاف أداة القائمة التفصيلية إلى لوحة التحكم.
3. إذا لم تُوجد، تُنشأ أداة قائمة جديدة ديناميكياً مع:
   - العنوان: `{عنوان الأداة الأصل} - {تسمية الشريحة}`
   - نفس مصدر البيانات والتجميع للأداة الأصل
   - `filterValue` مضبوط على تسمية الشريحة المنقورة
   - موضوعة أسفل الأداة الأصل في الشبكة
4. يحترم التنقل التفصيلي حد الأدوات — إذا تم بلوغ الحد، يُعرض تحذير.

## وضع تحرير التخطيط

عند النقر على **تحرير التخطيط**:

1. تظهر أدوات السحب (فئة `.drag-handle`) على يسار كل أداة.
2. يمكنك سحب البطاقات إلى مواضع جديدة على شبكة الـ 12 عموداً.
3. يمكنك تغيير حجم البطاقات من أي زاوية (SE، SW، NE، NW).
4. يُحفظ تخطيطك تلقائياً في PostgreSQL (`user_preferences.settings.dashboards`) عند الإفلات أو تغيير الحجم.
5. انقر **تحرير التخطيط** مرة أخرى لإيقافه وقفل التخطيط.

### تفاصيل تخطيط الشبكة

- **الشبكة**: `react-grid-layout` مع `WidthProvider` لعرض متجاوب.
- **الأعمدة**: 12 (ثابت).
- **ارتفاع الصف**: 64 بكسل.
- **الهوامش**: 12 بكسل بين العناصر.
- **الضغط**: لا يوجد (`compactType={null}`) — تبقى الأدوات حيث وُضعت.
- **منع التصادم**: معطل (`preventCollision={false}`).
- **دعم RTL**: تُعكس مواضع الشبكة لتخطيطات RTL. تبقى حاوية الشبكة LTR للتموضع، لكن محتوى الأداة يكون RTL.

### ضغط التخطيط المصفى

عندما يكون بحث أو مرشح فئة نشطاً، تُعاد حزم الأدوات المرئية من أعلى اليسار باستخدام دالة `compactLayoutItems` مخصصة لتجنب الفجوات.

## استمرارية الأدوات

تُحفظ الأدوات والتخطيطات **لكل مستخدم** في PostgreSQL عبر خطاف `useWidgetDashboard`. مسار التخزين هو `user_preferences.settings.dashboards[storageKey]`، حيث يُعرّف `storageKey` سياق لوحة التحكم (مثل `dashboard_analytics`). لا تؤثر تخصيصاتك على المستخدمين الآخرين.

### حد الأدوات

لكل لوحة تحكم حد أقصى لعدد الأدوات (`DASHBOARD_ANALYTICS_MAX_WIDGETS`)، مضبوط على عدد الأدوات الافتراضية. عند بلوغ الحد:

- يعرض زر **إضافة أداة** لافتة تحذير: "تم بلوغ حد الأدوات — الحد الأقصى N أداة (افتراضي النظام). احذف أو حرر الأدوات الموجودة قبل إضافة المزيد."
- يُلغي التحذير تلقائياً بعد 6 ثوانٍ.
- إجراءات التكرار والتنقل التفصيلي تحترم هذا الحد أيضاً.

## إعادة التعيين إلى الافتراضيات

انقر أيقونة **إعادة التعيين** (التاريخ) لفتح نافذة تأكيد:

> **إعادة تعيين لوحة التحكم** — إعادة تعيين لوحة التحكم إلى الافتراضيات؟ سيتم فقدان التخطيط المخصص الخاص بك.

عند التأكيد:
- تُزال جميع الأدوات المخصصة.
- تُستعادة مجموعة الأدوات الافتراضية.
- تُمسح حالات التصغير والأحجام الأصلية وإصدارات التحديث والطوابع الزمنية.
- تُحفظ إعادة التعيين في PostgreSQL.

## معالجة البيانات

تُعالج بيانات الأدوات بواسطة دالة `processWidgetData`، والتي:

1. تقرأ `dataSource` و`groupBy` و`aggregation` و`filters` و`dateRange` و`customDateFrom` و`customDateTo` و`valueField` للأداة.
2. تسترجع مصفوفة البيانات الخام من كائن `rawData`.
3. تطبق المرشحات العامة (programId، subjectId، classId، إلخ).
4. تطبق مرشحات مستوى الأداة.
5. تطبق تصفية النطاق الزمني (اليوم، آخر 7/30/90 يوماً، نطاق مخصص).
6. تجمع البيانات حسب الحقل المحدد.
7. تجمع باستخدام العدد أو المجموع أو المتوسط.
8. تُرجع مصفوفة من أزواج `{ label, value }` مناسبة لعرض الرسوم البيانية.

لمصادر النظرة العامة/العد (مثل `driveOverview` و`workflowOverview` و`activityOverview`)، تستخرج الدالة مفتاح الإحصائية المحدد من كائن النظرة العامة. لـ `totalStorageSize`، تُحوّل القيمة من بايت إلى ميجابايت.

## خدمات واجهة برمجة التطبيقات

يوفر `DashboardAnalyticsService` أربع نقاط وصول:

- **تحليلات عامة** — يجلب بيانات تحليلية عامة.
- **تحليلات القرص الذكي** — يجلب تحليلات خاصة بالقرص الذكي (نظرة عامة، ملفات حسب النوع/الدلو، نشاطات، تخزين حسب المستخدم، ملفات حديثة).
- **تحليلات سير العمل** — يجلب تحليلات خاصة بسير العمل (نظرة عامة، حسب الحالة/النوع/البرنامج، خط زمني، مستندات حديثة).
- **تحليلات الأنشطة** — يجلب تحليلات خاصة بالأنشطة (نظرة عامة، أنشطة حسب النوع، خط زمني، إرسالات حسب الحالة، خط زمني للإرسالات، موارد حسب النوع، أنشطة/إرسالات حديثة).

جميع استدعاءات واجهة البرمجة تتضمن معالجة الأخطاء وتُرجع البيانات بالتنسيق المتوقع من `buildAnalyticsRawData`، الذي يعيّن استجابة واجهة البرمجة إلى بنية `rawData` التي تستهلكها `processWidgetData`.

## التحققات والقيود

- **حد الأدوات**: كل لوحة تحكم تطبق حدًا أقصى لعدد الأدوات. الإضافة أو التكرار أو التنقل التفصيلي بعد بلوغ الحد يُمنع مع تحذير.
- **توافق نوع الرسم**: لا تدعم جميع مصادر البيانات جميع أنواع الرسوم. الخيارات غير المتوافقة معطلة في المنشئ.
- **أدوات العدد**: لا تحتوي على خيارات تجميع أو تجميع حسب — تعرض مقياساً رقمياً واحداً محسوباً مسبقاً.
- **مصادر القائمة**: فقط مصادر بيانات محددة تدعم أدوات القائمة (مُعرّفة في `LIST_ALLOWED_SOURCES`).
- **نطاق التاريخ**: النطاق المخصص يتطلب تعيين تاريخ البداية والنهاية معاً.
- **تخطيط RTL**: يتم عكس مواضع الشبكة لـ RTL، لكن حاوية الشبكة تبقى LTR للتموضع الصحيح.
- **أخطاء الصلاحيات**: إذا أرجعت خدمة بيانات `permission-denied`، تعرض البطاقة `0` ويُسجل الخطأ.

## استكشاف الأخطاء وإصلاحها

| المشكلة | السبب | الحل |
| --- | --- | --- |
| الأداة تعرض 0 أو فارغة | فشل خدمة البيانات أو أرجأت فارغاً | انقر **تحديث** لإعادة جلب البيانات. تحقق من وحدة التحكم في المتصفح من أخطاء الصلاحيات. |
| تعذّر إضافة أداة | تم بلوغ حد الأدوات | احذف أو حرر الأدوات الموجودة قبل إضافة المزيد. |
| نوع الرسم معطل | لا توجد مصادر بيانات متوافقة مع نوع الرسم في الفئة المختارة | بدّل إلى فئة أو نوع رسم مختلف. |
| التخطيط لا يُحفظ | فشل حفظ تفضيلات المستخدم في PostgreSQL | تحقق من اتصال الشبكة وسجلات الخادم. تُحفظ تغييرات التخطيط عبر خطاف `useWidgetDashboard`. |
| الجولة لا تبدأ | الجولة شوهدت بالفعل (محفوظة في `localStorage`) | انقر **أيقونة المساعدة** (?) في رأس القسم لإعادة تشغيل الجولة. |
| التنقل التفصيلي لا يعمل | تم بلوغ حد الأدوات أو لا يوجد مصدر تنقل تفصيلي متوافق | تحقق مما إذا كنت عند حد الأدوات. قد لا يكون لبعض المصادر أدوات تنقل تفصيلي محددة مسبقاً. |
| أداة العدد تعرض رقماً خاطئاً | عدم تطابق مفتاح الإحصائية أو عدم تحميل البيانات | تحقق من صحة مصدر البيانات والمقياس. حدّث لوحة التحكم. |
| النطاق المخصص لا يُطبق | تاريخ البداية أو النهاية مفقود | تأكد من تعيين كلا التاريخين في `DateRangeSlider`. |
